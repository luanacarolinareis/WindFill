Troubleshooting

If autofill is not working, use this page to narrow it down quickly before changing a lot of settings.

Quick rule

If the popup says no match, fix the pattern. If it matches but does not fill, add selectors.

Most issues are either pattern matching or unusual field selectors.
Theme

The popup says no matching profile

Check the profile pattern first. The pattern must match the real page URL, not just the IP you remember.

  • Exact page: http://192.168.1.130/login
  • All pages on one IP: http://192.168.1.130/*
  • Host wildcard only: 10.0.0.*

The profile matches, but fields stay empty

The page probably uses unusual field names. Add selectors for the username and password fields.

  • Good selectors: #username
  • Good selectors: input[name="operator-code"]
  • Avoid brittle selectors like div > form > input:nth-child(2)

The popup says the page is blocking extension scripts

This usually happens when the controller tab was already open before the extension was reloaded, so the page script needs to be attached again.

  • Open chrome://extensions
  • Click Reload on the extension
  • Go back to the controller page and refresh it
  • Open the popup again and test once more

Auto submit does not work

If filling works but the page does not continue, add a submit selector for the real login button.

  • Examples: #loginBtn
  • Examples: button[type="submit"]
  • Save the profile and refresh the page after changing it

How to find selectors in Chrome

Right-click the field, choose Inspect, then look for stable attributes like id or name.

  • Username examples: #user, input[name="username"]
  • Password examples: #pwd, input[type="password"]
  • Button examples: #loginBtn, button[type="submit"]

Quick debug checklist

  • Is the profile enabled?
  • Did you click save?
  • Did you refresh the target page after saving?
  • Does the popup show a matching profile?
  • If it matches, do the fields need selectors?
